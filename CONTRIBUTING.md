# Contributing to DEHUB

First off, thank you for considering contributing to DEHUB! It's people like you that make DEHUB such a great resource for the data engineering community.

## Code of Conduct

This project and everyone participating in it is governed by the [DEHUB Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs

- Use the GitHub Issues tracker
- Describe the issue clearly
- Include steps to reproduce

### Suggesting Resources

We welcome suggestions for:
- 📚 Books and courses
- 🛠️ Tools and frameworks  
- 🌐 Communities and conferences
- 🎙️ Podcasts and newsletters
- 💼 Project ideas

### Pull Request Guidelines

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/new-resource`
3. Make your changes following the existing format
4. Commit with a clear message: `git commit -m "Add: [Resource Name] to [Category]"`
5. Push to your fork: `git push origin feature/new-resource`
6. Open a Pull Request with a clear description

### Format Guidelines

When adding resources, follow this format:

```markdown
| [Resource Name](link) | Brief description | Level/Status |
```

For tools, include the GitHub stars badge when available:
```markdown
| [Tool Name](link) | ![Stars](https://img.shields.io/github/stars/org/repo?style=flat-square) | Description |
```

## What Makes a Good Resource?

- **Relevant**: Directly applicable to data engineering
- **High Quality**: Well-maintained, accurate, and valuable
- **Accessible**: Preferably free or reasonably priced
- **Active**: Not abandoned or severely outdated

## Getting Help

- Open an Issue with your question
- Join our community on Discord (link in README)

Thank you for your contribution! 🚀
